ingest

Systematically retrieve paywalled and hard-to-access articles for research — with a tested strategy cascade and structured output

Add to Claude Code
$ cp ingest.md .claude/commands/ingest.md

A Claude Code skill that retrieves article content using a cascade of eight bypass strategies — working from direct fetch through syndication hunting — and saves structured markdown source files with metadata, key facts, and access status.

  • Eight-strategy cascade tested live against real URLs (Apr 2026)
  • Dead services removed: 12ft.io (shut down July 2025), removepaywall.com, Google Cache
  • Outputs structured source files with frontmatter, key facts, and cross-references
  • Designed for long-form investigative research — builds a citable source library

How it works

Give Claude Code a URL. The skill walks through eight access strategies in order, stopping at the first one that returns full article text. It then formats the content as a structured markdown source file: title, author, publication, date, URL, access status, summary, key facts for sourcing, and cross-references to related files.

The skill is designed for investigative research where you're building a citable library across dozens or hundreds of sources. Every file it produces is meant to be read again — by you or by Claude Code — with full provenance intact.

If the first strategy fails and you're present in the conversation, it stops and asks before proceeding. It never silently reconstructs from secondary sources or buries an access failure in metadata.

Strategy cascade

Strategies are tried in order. Each is marked with its test status from the April 2026 live test session.

  • 1
    Direct fetch
    WebFetch the original URL. Works for Substack, PBS, NPR, ProPublica, and most open-access outlets.
    Working Fails for Bloomberg, NYT, WSJ, Cloudflare-protected sites
  • 2
    Jina AI Reader
    Prepend r.jina.ai/ to the URL. Returns clean Markdown with title, author, date, and body. Renders pages in a headless browser before extraction — handles soft/metered paywalls.
    Working Not a hard paywall bypass — serves whatever the server returns. Fails on Bloomberg (403 + bot detection)
  • 3
    Wayback Machine CDX + fetch
    First check the CDX API for captures, then fetch the most recent archived version. CDX pre-check prevents wasted fetches on uncrawled URLs.
    Working Only works if Wayback crawled the page. CDX check is fast and free
  • 4
    archive.ph / archive.today
    Fetch archive.ph/newest/{URL}. Returns a snapshot if someone previously archived the page manually.
    Conditional Returns 404 on recent or niche articles that were never manually archived. Programmatic submission requires CAPTCHA bypass — not viable
  • 5
    Syndication hunt
    Search for the article on known syndication platforms. Bloomberg articles often appear on BritBrief or Yahoo Finance. NYT syndicates to Yahoo News, AOL, MSN. WSJ to Yahoo Finance.
    High success rate Often the fastest path for major outlet paywalls — try before giving up
  • 6
    AMP / mobile version
    Some sites serve less-restricted mobile versions. Try amp.{domain}/{path}, {domain}/amp/{path}, or the Google AMP cache.
    Conditional Hit or miss — depends on whether the site serves AMP and whether it respects access differently
  • 7
    Ask the user
    If strategies 1–6 fail and you're present in the conversation, the skill stops here and asks: "I couldn't get the full text from [domain]. Do you have access? Can you paste it?"
    Stop and ask Never skipped when user is present. User-pasted content = full access
  • 8
    Secondary source reconstruction
    Only with explicit user permission. Reconstruct key facts and quotes from other outlets that covered the same story.
    Last resort Output is marked PARTIAL. Never used silently. Only for dead links or content nobody can access

Hard limits

Some sites have no automated bypass. Known from live testing:

Site Blocker Best path
Bloomberg 403 + bot detection on all automated clients Syndication (BritBrief, Yahoo Finance) or user paste
NYT Hard paywall Syndication (Yahoo News, AOL, MSN) or user paste
WSJ Hard paywall Syndication (Yahoo Finance) or user paste
Cloudflare Enterprise TLS fingerprinting before JS runs Ask user

For Bloomberg specifically: nothing automated retrieved content in testing. Syndication (BritBrief, Yahoo Finance) is the only reliable programmatic path.

Output format

Each ingested article is saved as a markdown file with this structure:

# Article Title

**Source:** Publication Name
**Author:** Author Name
**Date:** Month Day, Year
**URL:** https://original-url
**Access:** DONE — Strategy 2 (Jina AI Reader)
**Retrieved:** 2026-04-23

---

## Summary
2–3 sentence summary of main findings.

## Key Content
Full text or organized sections of key quotes and facts.

## Key Facts for Sourcing
- **Fact** — context and significance
- **"Quote"** — who said it, when

## Cross-References
- See: related-source.md (brief description)

Access status is one of DONE (full text), PARTIAL (reconstructed from secondary sources), or DEAD (link is gone, only snippets recovered). This is always honest — DONE means you actually got the full text.

Behavior rules

Strategy 1 fails — what happens?

If the user is present in the conversation, the skill stops and asks before proceeding to secondary strategies. It does not silently continue. The only exception is --no-save mode or when the user has explicitly said to try all bypass strategies.

User pasted the article in the conversation.

That content is treated as full access. No fetching, no reconstruction — the skill extracts the text directly from the conversation and treats it as DONE.

What does the skill never do?

Never fabricate content. Never bury an access failure in metadata. Never reconstruct from secondary sources without explicit permission. Never mark a file DONE when it's actually PARTIAL.

How many strategies before giving up?

At least 3 before stopping. The full cascade exists because different strategies work for different sites — direct fetch may fail where syndication succeeds.
