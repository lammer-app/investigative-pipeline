investigative-pipeline

Turn research documents into a queryable evidence database

Install
$ git clone https://github.com/luxwon/investigative-pipeline

A Python toolkit for turning books, articles, depositions, and transcripts into a searchable evidence database — with search, timeline, and SQLite query tools on top.

  • No external dependencies — Python 3.11+ stdlib only
  • Everything is plain files you can grep, diff, and edit
  • Handles EPUB, PDF, Kindle, articles, YouTube, and audio
  • JSONL during active research; SQLite when the corpus is stable

Overview

You're deep in a research project with dozens of books, hundreds of articles, and scattered transcripts. You want to find every claim about a specific person across all sources, build a chronological timeline, and surface connections between entities.

This does that. Point it at a markdown file, it scores sentences for factual density, extracts the keepers into JSONL, and gives you search, timeline, and SQLite query tools on top.

The pipeline accumulates across sources — 1 document or 1,000, same workflow. Set PIPELINE_VAULT and PIPELINE_ARCHIVE, add a source TOML per document, run extract_facts.py, search with search_facts.py.

Getting Started

Requirements: Python 3.11+. No pip installs needed.

1 — Configure paths

# Tell the pipeline where your project lives
export PIPELINE_VAULT=/Users/you/projects/dark-money
export PIPELINE_ARCHIVE=/Users/you/projects/dark-money/data

# Or put both lines in a .env file at your project root
echo 'PIPELINE_VAULT=/Users/you/projects/dark-money' >> .env

# Verify, then create the directory structure
python3 src/project_paths.py
mkdir -p $PIPELINE_VAULT/{books,sources,extracts,config/sources}
mkdir -p $PIPELINE_ARCHIVE/{facts,views}

2 — Add a source config

One .toml per source. The filename stem must match the book/source file.

# config/sources/dark-money.toml  ←  maps to books/dark-money.md
cat > $PIPELINE_VAULT/config/sources/dark-money.toml << 'EOF'
[source]
id             = "SRC-BOOK-mayer-dark-money"
ref            = "Mayer, Jane. Dark Money (2016)"
type           = "book"
author         = "Jane Mayer"
year           = 2016
certainty_base = 8
is_memoir      = false
EOF

3 — Add entity aliases (optional)

# config/entities.toml — normalizes name variants in search
cat > $PIPELINE_VAULT/config/entities.toml << 'EOF'
[people.koch-charles]
canonical = "Charles Koch"
aliases   = ["C. Koch", "Charles de Ganahl Koch"]

[orgs.koch-industries]
canonical = "Koch Industries"
aliases   = ["Koch Industries Inc.", "Koch Ind."]
EOF

4 — Convert source and extract facts

# EPUB (recommended — see Ingestion section for other formats)
pandoc ~/Downloads/dark-money.epub -t markdown \
  -o $PIPELINE_VAULT/books/dark-money.md

# Extract facts from the converted file
python3 src/extract_facts.py $PIPELINE_VAULT/books/dark-money.md
# → [dark-money] Scanning 6,291 sentences...
# → [dark-money] After dedup: 394 facts
# → Done. Total facts in store: 394

python3 src/build_registry.py
python3 src/validate.py
# → 394 facts validated, 0 errors

5 — Search and query

python3 src/search_facts.py "Koch brothers" --full
python3 src/search_facts.py --person "Charles Koch" --year 2010
python3 src/search_facts.py --person "Charles Koch" --json > charles-koch.json

# When corpus is stable: migrate to SQLite and build timeline
python3 src/migrate_to_sqlite.py
python3 src/build_timeline.py --full

Ingestion Pathways

The extractor reads plain markdown. Convert your source to .md first using the method below that matches your format.

EPUB

Use this whenever available. Pandoc extracts structured text — 2–10x more content than a PDF scan.

pandoc ~/Downloads/book.epub -t markdown -o $PIPELINE_VAULT/books/book.md

# If pandoc fails with "parseSpine" error, the EPUB uses raw HTML files.
# Unzip and convert the HTML chapters instead:
unzip book.epub -d /tmp/book_extracted/
cat /tmp/book_extracted/OEBPS/chapter*.html > /tmp/combined.html
pandoc /tmp/combined.html -t markdown -o $PIPELINE_VAULT/books/book.md

# If HTML extraction left "--- Page N ---" markers that cause 0 facts:
python3 -c "
import re, sys
t = open(sys.argv[1]).read()
t = re.sub(r'--- Page \d+ ---\n', '', t)
open(sys.argv[1], 'w').write(t)
" $PIPELINE_VAULT/books/book.md

PDF (text-based)

# brew install poppler
pdftotext -layout source.pdf $PIPELINE_VAULT/books/source.md
# If output is garbage characters, it's a scanned PDF — use OCR below

PDF (scanned)

# brew install tesseract
tesseract source.pdf source txt
cp source.txt $PIPELINE_VAULT/sources/source.md

# Multi-page: convert to images first
pdftoppm -r 300 source.pdf /tmp/pages/page
for f in /tmp/pages/*.ppm; do tesseract "$f" "${f%.ppm}" txt; done
cat /tmp/pages/*.txt > $PIPELINE_VAULT/sources/source.md

Kindle

Use kindle-ai-export — Playwright screenshots Kindle Web Reader and extracts text via GPT-4 vision. Log into reads.amazon.com first.

git clone https://github.com/transitive-bullshit/kindle-ai-export
cd kindle-ai-export && npm install

# Find the ASIN (B0... code) in the Amazon book URL
npx kindle-ai-export --asin B09XXXXXXXX --output /tmp/book.md
cp /tmp/book.md $PIPELINE_VAULT/books/book.md

# Timeout? Add --timeout 60000. Session expired? Re-login to reads.amazon.com

Web article

# Open-access pages — pandoc fetches and converts directly:
pandoc "https://example.com/article" -t markdown \
  -o $PIPELINE_VAULT/sources/article-slug.md

# Blocked by user-agent? Use curl first:
curl -sL -A "Mozilla/5.0" "https://example.com/article" | \
  pandoc -f html -t markdown -o $PIPELINE_VAULT/sources/article-slug.md

# Paywalled: open in browser reader mode, select all, paste into .md
# Set type = "article" in the TOML. Articles go in sources/, not books/

YouTube video

# brew install yt-dlp
yt-dlp --write-auto-sub --sub-lang en --skip-download \
  --output "/tmp/%(title)s" "https://youtube.com/watch?v=VIDEO_ID"

# Convert VTT to clean text (strip timecodes, deduplicate lines):
python3 -c "
import re, sys
lines = open(sys.argv[1]).readlines()
seen, out = set(), []
for line in lines:
    line = line.strip()
    if '-->' in line or not line or line.startswith('WEBVTT'): continue
    line = re.sub(r'<[^>]+>', '', line)
    if line and line not in seen:
        seen.add(line); out.append(line)
print('\n'.join(out))
" /tmp/video.en.vtt > $PIPELINE_VAULT/sources/video-title.md

Audio / deposition / podcast

# Option A: Whisper — local, free, no API key
# pip install openai-whisper && brew install ffmpeg
whisper audio.mp3 --model medium --output_format txt --output_dir /tmp/
cp /tmp/audio.txt $PIPELINE_VAULT/extracts/deposition-name.md

# Option B: AssemblyAI — API, speaker diarization
# pip install assemblyai
python3 << 'EOF'
import assemblyai as aai
aai.settings.api_key = "YOUR_KEY"
t = aai.Transcriber().transcribe("audio.mp3",
    aai.TranscriptionConfig(speaker_labels=True))
with open("$PIPELINE_VAULT/extracts/name.md", "w") as f:
    for utt in t.utterances:
        f.write(f"[Speaker {utt.speaker}] {utt.text}\n\n")
EOF

# Set type = "deposition" or "transcript". Depositions go in extracts/

How It Works

  books/           sources/          extracts/
  (EPUB→pandoc)    (articles)        (depositions, transcripts)
       │                │                    │
       └────────────────┼────────────────────┘
                        ▼
              extract_facts.py
              sentence scoring (FACTUAL_INDICATORS regex)
              entity alias resolution (config/entities.toml)
              near-duplicate deduplication (Jaccard)
                        │
              ┌─────────┴──────────┐
              ▼                    ▼
       facts.jsonl           sources.jsonl
       media.jsonl           (build_registry.py)
              │
              ├── validate.py          schema checks + cross-reference
              ├── generate_views.py    markdown views by year/entity/theme
              ├── migrate_to_sqlite.py
              │         │
              │    evidence.db         13-table relational schema
              │         │
              ├── search_facts.py      entity / tag / theme queries
              └── build_timeline.py   chronological timeline.md

The extraction engine scores each sentence against FACTUAL_INDICATORS — regex patterns for years, dollar amounts, named entities, action verbs, and attribution phrases. Sentences above the score threshold (default: 5) become fact candidates. Near-duplicates are removed via Jaccard similarity.

Two phases: JSONL during active research (fast to append, easy to grep), SQLite when you're ready for richer querying. Re-migrate at any time as sources accumulate.

Scripts

Script What it does When to run
extract_facts.py Scores sentences, extracts facts to facts.jsonl After each new source
build_registry.py Builds sources.jsonl from config/sources/*.toml After adding or editing a TOML
validate.py Schema integrity and cross-reference checks After any extraction; --fix to auto-repair
generate_views.py Markdown views by year, entity, theme For browsable summaries
migrate_to_sqlite.py Loads JSONL into relational SQLite database When corpus is stable; safe to re-run
search_facts.py CLI query: entity, tag, theme, AI filters Anytime; works from JSONL or SQLite
build_timeline.py Chronological timeline.md (4-agent pass) After SQLite migration
project_paths.py Prints resolved paths Troubleshooting path issues

All scripts accept --help. Paths read from PIPELINE_VAULT and PIPELINE_ARCHIVE env vars, or from a .env file at your project root.

FAQ

Does the source TOML filename have to match the book filename?

Yes. The stem of the config file maps to the source file. config/sources/dark-money.toml maps to books/dark-money.md. Mismatched names cause silent metadata skips.

Extraction ran but I got 0 facts.

1. Empty input — check wc -l books/source.md. If near zero, conversion failed silently. 2. Threshold too high — try --threshold 3. 3. Page marker artifacts from HTML EPUB extraction — strip --- Page N --- lines (see Ingestion section).

pandoc fails with a parseSpine error on my EPUB.

Some EPUBs use raw HTML page files instead of the standard spine structure. Unzip the EPUB and convert the HTML chapters directly — see the EPUB ingestion section above.

How do I map name variants to one entity?

# config/entities.toml
[people.smith-jane]
canonical = "Jane Smith"
aliases   = ["J. Smith", "Jane A. Smith", "Smith, Jane"]

The extractor resolves all aliases to the canonical name. Re-run extraction after updating entities.toml to apply changes.

JSONL vs SQLite — when do I need to migrate?

Stay in JSONL while actively ingesting new sources. Migrate to SQLite when you need entity cross-queries, timeline generation, theme bucketing, or AI queries via search_facts.py --ai. The SQLite database is fully rebuilt from JSONL each time — re-migrating is safe and normal.

What does certainty mean?

A 0–10 scale set per source in its TOML (certainty_base), adjustable per-fact in facts.jsonl. 8–10: court documents, sworn testimony. 5–7: credible journalism with named sources. 1–4: unverified, single anonymous source. Set conservatively, raise per-fact when corroborated.

How do I add a second source and keep facts separate?

Add a new TOML, point extraction at the new file, re-run build_registry + validate. Each fact's sources array carries its source ID. Facts accumulate in the same facts.jsonl.

python3 src/extract_facts.py books/second-source.md
python3 src/build_registry.py && python3 src/validate.py
# Total facts in store: 812  (394 + 418)

Data Formats

All fact data is newline-delimited JSON (.jsonl) — one object per line, grep and diff friendly. The format is stable; migrate to SQLite at any point for relational queries.

{
  "id":            "F-2016-0042",
  "claim":         "Charles Koch donated $2.5M to Club for Growth in Q3 2016.",
  "date":          { "value": "2016", "precision": "year" },
  "certainty":     8,
  "status":        "confirmed",
  "sources": [
    { "source_id": "SRC-BOOK-mayer-dark-money", "page": "142" }
  ],
  "people":        ["Charles Koch"],
  "places":        [],
  "organizations": ["Club for Growth"],
  "tags":          ["financial", "political"],
  "created":       "2025-01-15"
}
FieldDescription
idF-{YEAR}-{NNNN} — auto-assigned from event date
claimSingle factual statement. Edit manually to clean up artifacts.
date.precisionexact | month | year | range | undated | approximate
certainty0–10. Inherited from source certainty_base. Adjust per-fact.
statusalleged | confirmed | disputed | disproven
sourcesArray of {source_id, page, quote}. source_id matches sources.jsonl.
people/places/orgsEntity arrays — canonical names from entities.toml
